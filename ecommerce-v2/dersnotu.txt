PROPS, PROP DRILLING VE CONTEXT API DERS NOTLARI

=========================================
SABAH BÖLÜMÜ: COMPONENTLER ARASI VERİ AKTARIMI VE PROP DRILLING
=========================================

1. BILEŞENLER ARASI VERİ AKIŞI
React'te veri akışı yukarıdan aşağıya (Ata bileşenden Yavru bileşene) tek yönlü olarak gerçekleşir:
- Parent to Child: Props aracılığıyla veri veya fonksiyonlar aşağı aktarılır.
- Child to Parent: Child bileşen, kendisine prop olarak geçilen bir callback fonksiyonunu çağırarak Parent bileşene veri gönderebilir veya durumu tetikleyebilir.


2. PROP DRILLING (PROP DELME) PROBLEMİ NEDİR?
Bir state verisinin, ağacın en üstündeki bileşenden en altındaki bileşene aktarılabilmesi için aradaki "veriye ihtiyaç duymayan" tüm bileşenlerden prop olarak geçirilmesi durumudur.
Sorunları:
- Kod Kirliliği: Bileşenlerin parametre listeleri gereksiz proplarla dolup taşar.
- Bakım Zorluğu: Araya yeni bir bileşen girdiğinde veya veri ismi değiştiğinde tüm zincirin güncellenmesi gerekir.
- Sıkı Bağlılık (Tight Coupling): Bileşenlerin bağımsız olarak tekrar kullanılabilmesini zorlaştırır.


=========================================
ÖĞLEDEN SONRA BÖLÜMÜ: CONTEXT API VE GLOBAL STATE YÖNETİMİ
=========================================

1. CONTEXT API NEDİR?
React'te prop drilling yapmadan, bileşen ağacının herhangi bir seviyesindeki bileşenlerin doğrudan erişebileceği global veri depoları oluşturmamızı sağlayan yerleşik bir mekanizmadır.
3 ana aşamada kurulur:
- Context Oluşturma (`createContext`): Global deponun kendisini tanımlar.
- Provider (Sağlayıcı)     : Verileri tüm çocukların erişimine sunan kapsayıcıdır.
- Consumer (Tüketici)      : Veriyi okuyan alt bileşenlerdir (`useContext` hook'u ile).


2. CREATECONTEXT VE PROVIDER YAPILANDIRMASI
Öncelikle bir Context dosyası (`CartContext.js`) oluşturulur ve dışarıya aktarılır:
import { createContext } from 'react';
export const CartContext = createContext();

Ardından ana sarmalayıcı (genellikle `App.jsx` veya bir `CartProvider.jsx`) içinde Provider konumlandırılır ve `value` prop'u ile paylaşılacak state/metotlar geçilir:
<CartContext.Provider value={{ sepet, sepetEkle, sepetCikar }}>
    <Navigasyon />
    <UrunListesi />
</CartContext.Provider>


3. USECONTEXT HOOK İLE TÜKETİM
Alt bileşenlerde (örneğin sepet ikonunda veya ürün kartında) verileri okumak için `useContext` kancası kullanılır:
import { useContext } from 'react';
import { CartContext } from './CartContext';

const SepetIkonu = () => {
    const { sepet } = useContext(CartContext); // prop drilling olmadan doğrudan erişim
    return <span>{sepet.length}</span>;
};


4. CONTEXT API İLE GLOBAL TEMA VEYA SEPET YÖNETİMİ
Projede Context API'nin en sık kullanıldığı alanlar:
- Alışveriş Sepeti (Cart Context): Herhangi bir ürün kartından sepete ekleme yapıldığında sepet sayacının ve sepet sayfasının otomatik güncellenmesi.
- Tema Yönetimi (Theme Context): Tüm sitenin "Karanlık Mod (Dark Mode)" ile "Aydınlık Mod" arasında tek tıkla geçiş yapabilmesi ve tüm alt bileşenlerin bu temaya göre renk değiştirmesi.
- Kullanıcı Oturumu (Auth Context): Giriş yapan kullanıcının profil bilgilerinin ve yetkilerinin tüm sayfalarda denetlenmesi.


=========================================
ÖZET TABLO: HANGİ KAVRAM NE İŞE YARAR?
=========================================

1. CONTEXT API BİLEŞENLERİ:
- createContext()          : Yeni bir global veri alanı (Context) oluşturan React fonksiyonu.
- Provider                 : Oluşturulan context'i alt bileşenlere sunan ve güncel verileri dağıtan sağlayıcı bileşen.
- value                    : Provider bileşeninin altındaki tüm ağaca dağıtacağı verileri/fonksiyonları barındıran prop.
- useContext()             : Belirtilen context'teki güncel verileri okumayı sağlayan fonksiyonel kanca.

2. VERİ AKTARIMI KAVRAMLARI:
- Tek Yönlü Veri Akışı      : Verilerin React'te sadece yukarıdan aşağıya akması kuralı.
- Prop Drilling            : Verilerin aradaki ilgisiz bileşenler üzerinden en alta kadar taşınması problemi.
- Global State             : Uygulamanın her yerinden erişilebilen ve paylaşılan merkezi durum yönetimi.

=========================================
TEMEL UYGULAMA VE PRATİK KAZANIMLARI
=========================================

1. PROP DRILLING YAŞANTI DENEYİMİ:
- TrendStore v2 projesindeki `sepet` state'inin `App`'ten `Header`'a, oradan da `SepetGezgini`'ne taşınması esnasındaki prop kalabalığı simüle edildi.

2. CONTEXT OLUŞTURMA VE PROVIDER KURULUMU:
- Alışveriş sepeti verileri için `CartContext` tanımlandı.
- Tüm ürün listesini ve header bileşenini kapsayacak şekilde `CartContext.Provider` entegre edildi.

3. GLOBAL SEPET ETKİLEŞİMİ:
- `ProductCard` bileşeni içindeki "Sepete Ekle" butonunun, props'lardan bağımsız olarak doğrudan `useContext(CartContext)` kullanarak merkezi sepet durumunu güncellemesi sağlandı.
- Arayüzde hiçbir ara prop geçişine gerek kalmadan, sepet toplam fiyatı ve ürün sayılarının eşzamanlı güncellendiği doğrulandı.

=========================================
PROJEDE KULLANILAN METOT VE HOOK'LARIN AÇIKLAMALARI
=========================================

- **createContext:** Boş veya varsayılan değerli bir Context nesnesi üretir.
  *Kullanımı:* `const CartContext = createContext(null)` ile sepet yönetim veri alanı tanımlamada.

- **useContext:** Belirtilen Context nesnesinin Provider tarafından sağlanan en güncel değerini döndürür.
  *Kullanımı:* Alt kart bileşenlerinde global sepet fonksiyonlarını (`sepetEkle` vb.) çağırabilmede.

- **CartContext.Provider:** Verilerin dağıtımını üstlenen kapsayıcıdır.
  *Kullanımı:* `App.jsx` içinde tüm alt bileşen ağacını sarmalayarak sepet verilerine erişim izni vermede.

- **value={{ cart, addToCart }}:** Provider'ın çocuklara sunduğu nesnedir.
  *Kullanımı:* Sepet dizisi ile sepet ekleme fonksiyonunu global erişime açmada.
